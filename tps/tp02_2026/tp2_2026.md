## Dataset — Restaurantes pelo Mundo

Este dataset apresenta uma coleção diversificada de restaurantes distribuídos por diferentes cidades ao redor do mundo, refletindo a riqueza cultural e gastronômica de diversas regiões. A base reúne estabelecimentos com diferentes perfis, variando em capacidade, faixa de preço, tipos de cozinha e avaliações, permitindo uma visão ampla do cenário gastronômico global.

Através desses dados, é possível explorar padrões interessantes, como a relação entre localização e estilo culinário, variações de avaliação entre diferentes tipos de restaurantes e características operacionais como horário de funcionamento e tempo de atividade. Cada registro representa um estabelecimento único, contribuindo para um conjunto de dados rico e multifacetado.

Este dataset convida à investigação e análise, estimulando a descoberta de tendências, comparações entre cidades e o desenvolvimento de soluções que simulam aplicações do mundo real, como sistemas de busca, recomendação e organização de informações.

O dataset permite explorar conceitos importantes como organização e tipagem de dados, modelagem de entidades do mundo real, manipulação de estruturas (listas, registros, etc.), bem como consultas, filtragens e ordenações.

A tabela a seguir apresenta o dicionário de dados referente ao dataset. O dataset está disponível no arquivo `restaurantes.csv`.

### Dicionário de Dados

| Campo | Descrição | Tipo / Formato |
|---|---|---|
| `id` | Identificador único do restaurante (chave primária) | Inteiro (`int`) |
| `nome` | Nome do restaurante | String (texto ASCII) |
| `cidade` | Cidade onde o restaurante está localizado | String (texto ASCII) |
| `capacidade` | Número máximo de clientes que o restaurante comporta | Inteiro (`int`) |
| `avaliacao` | Nota média do restaurante (simulando avaliações de usuários) | Número real (`double`, ex: `4.5`) |
| `tipos_cozinha` | Tipos de culinária oferecidos (pode conter mais de um tipo) | Lista (string separada por `;`) |
| `faixa_preco` | Nível de preço do restaurante | String (`$`, `$$`, `$$$`, `$$$$`) |
| `horario` | Horário de funcionamento diário | String (`HH:mm-HH:mm`) |
| `data_abertura` | Data de abertura do restaurante | Data (`YYYY-MM-DD`) |
| `aberto` | Indica se o restaurante está em funcionamento atualmente | Booleano (`true` ou `false`) |

## Modelagem de Dados: Restaurante

Neste trabalho, você deverá realizar a modelagem de dados de um sistema que representa uma coleção de restaurantes distribuídos em diferentes cidades do mundo.

O objetivo é definir estruturas de dados que permitam representar adequadamente as informações de cada restaurante, servindo como base para etapas futuras do trabalho, nas quais serão aplicados algoritmos e estruturas de dados como listas, filas, árvores, além de operações de pesquisa e ordenação.

### Objetivo

Projetar e implementar os seguintes tipos de dados:

- `Restaurante`
- `Data`
- `Hora`

Esses tipos devem ser definidos de forma clara e organizada, permitindo representar fielmente os dados provenientes de um arquivo CSV contendo informações sobre restaurantes.

### Estilo de Codificação

Para a codificação, deve-se utilizar as seguintes convenções de nomenclatura:

- **Java:** camel case
- **C:** snake case

Tipos (`class` e `struct`) devem iniciar com letra maiúscula, enquanto variáveis e métodos (funções) devem iniciar com letra minúscula.

---

## Especificação dos Tipos

### Tipo `Data`

Representa uma data no calendário.

- `ano` (inteiro)
- `mes` (inteiro)
- `dia` (inteiro)

### Tipo `Hora`

Representa um horário do dia.

- `hora` (inteiro, de 0 a 23)
- `minuto` (inteiro, de 0 a 59)

### Tipo `Restaurante`

Representa um restaurante com suas características principais.

- `id`: identificador único (inteiro)
- `nome`: nome do restaurante (string)
- `cidade`: cidade onde está localizado (string)
- `capacidade`: capacidade máxima de clientes (inteiro)
- `avaliacao`: nota média (double)
- `tipos_cozinha`: coleção de tipos de culinária (arranjo/lista de strings)
- `faixa_preco`: nível de preço (inteiro de 1 a 4), onde:
  - `1` → `$`
  - `2` → `$$`
  - `3` → `$$$`
  - `4` → `$$$$`
- `horario_abertura`: horário de abertura (tipo `Hora`)
- `horario_fechamento`: horário de fechamento (tipo `Hora`)
- `data_abertura`: data de abertura (tipo `Data`)
- `aberto`: indica se o restaurante está em funcionamento (booleano)

### Regras de Mapeamento (CSV → Estruturas)

Ao ler o arquivo CSV:

- O campo `tipos_cozinha` deve ser convertido em um arranjo de strings, utilizando o separador `;`
- O campo `faixa_preco` deve ser convertido para um inteiro de 1 a 4
- O campo `horario` (formato `HH:mm-HH:mm`) deve ser dividido em:
  - `horario_abertura`
  - `horario_fechamento`
- O campo `data_abertura` (formato `YYYY-MM-DD`) deve ser convertido para o tipo `Data`
- O campo `aberto` deve ser interpretado como valor booleano

### Coleção de Restaurantes

Estrutura que representa uma coleção de restaurantes para armazenar os restaurantes lidos do dataset. Essa coleção deve ser modelada como um arranjo de ponteiros para `Restaurante` e será responsável por ler o arquivo CSV. Crie um tipo denominado `ColecaoRestaurantes` com os campos:

- `tamanho`: quantidade de itens (inteiro)
- `restaurantes`: arranjo de ponteiros para `Restaurante`

### Observações

- A modelagem deve ser compatível tanto com linguagem C quanto Java
- É vedada a dependência de bibliotecas externas
- Priorize clareza, organização e boa definição dos tipos
- Pense na reutilização das estruturas em etapas futuras

### Próximos Passos

Em etapas futuras, essa modelagem será utilizada para:

- leitura completa do dataset
- ordenação por diferentes critérios (nome, avaliacao, cidade)
- busca (sequencial e binária)
- uso de estruturas como pilhas, filas e árvores

## Estruturas Sequenciais

### 11. ☕ Lista com Alocação Sequencial em Java

Crie uma Lista de registros baseada na de inteiros vista na sala de aula. Sua lista deve conter todos os atributos e métodos existentes na lista de inteiros, contudo, adaptados para a classe `Restaurante`. Lembre-se que temos uma lista de ponteiros (ou referências) e cada um deles aponta para um registro. Neste exercício, faremos inserções, remoções e mostraremos os elementos de nossa lista.

Os métodos de inserir e remover devem operar conforme descrito a seguir:

| Método | Descrição |
|---|---|
| `void inserirInicio(Restaurante restaurante)` | Insere um registro na primeira posição da Lista e remaneja os demais |
| `void inserir(Restaurante restaurante, int posicao)` | Insere um registro na posição `posicao` da Lista, onde `posicao < n`; remaneja os demais |
| `void inserirFim(Restaurante restaurante)` | Insere um registro na última posição da Lista |
| `Restaurante removerInicio()` | Remove e retorna o primeiro registro cadastrado na Lista e remaneja os demais |
| `Restaurante remover(int posicao)` | Remove e retorna o registro cadastrado na posição informada da Lista e remaneja os demais |
| `Restaurante removerFim()` | Remove e retorna o último registro cadastrado na lista |

**Entrada padrão:** composta por duas partes:

- **Parte 1:** igual à entrada da primeira questão; registros inseridos ao final da Lista
- **Parte 2:** a primeira linha contém um inteiro `n` indicando a quantidade de operações; as próximas `n` linhas contêm comandos:

| Comando | Operação |
|---|---|
| `II` | Inserir no início |
| `I*` | Inserir na posição informada (posição logo após o comando) |
| `IF` | Inserir no fim |
| `RI` | Remover no início |
| `R*` | Remover na posição informada |
| `RF` | Remover no fim |

Nos comandos de inserção, temos também o identificador do registro a ser inserido.

**Saída padrão:** uma linha para cada registro removido, com `(R)` seguido do atributo `nome` do registro removido. No final, exibe os registros presentes na Lista do primeiro ao último.
